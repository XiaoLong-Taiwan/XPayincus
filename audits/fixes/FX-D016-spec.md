# FX-D016 规格：IPv6 CIDR 不检子网重叠、不限前缀(P1 / D-016 / M13-03)

## 根因
`server/src/routes/ip-addresses.ts:620` + `server/src/db/ipv6-subnets.ts:74`:自定义 IPv6 CIDR 只校验"是否含于宿主网段"与"字符串完全重复",不查**子网区间重叠**,也不限制前缀(如 /112,/120,/124)。→ 不同实例可能获重叠路由 → 流量串扰/路由劫持/大范围 IPv6 不可达。

## 修法(只改 ip-addresses.ts + db/ipv6-subnets.ts + 守卫;不碰 instances.ts(并行)、不改 package.json/schema)
1. 先只读:现有 CIDR 校验、宿主网段来源、已分配子网如何存/查、是否已有 IPv6 规范化/大整数工具。
2. **规范化 + 起止地址重叠检查**:
   - 把 CIDR 规范化为 [网络起始地址, 结束地址] 的 BigInt 区间(IPv6 128-bit)。
   - 校验落在宿主允许网段内(保留现有)。
   - 与**所有已分配子网**做区间重叠检查:任意 `startA<=endB && startB<=endA` 即拒(不再只比字符串相等)。
3. **强制允许前缀集合**:若代码/宿主配置已有支持前缀集则复用;否则强制 `{/112,/120,/124}`(按缺陷备注),超出集合→400。集合尽量取自现有常量,不擅自放宽。
4. 并发安全:分配时在事务内做重叠检查+插入,避免竞态双分配(能用唯一约束/事务就用)。

## 加守卫
在 ipv6/ip 相关现有守卫追加:重叠子网被拒、非法前缀被拒、规范化正确。不改 package.json。
