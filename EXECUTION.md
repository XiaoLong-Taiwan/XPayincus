# XPayincus 修复执行方案（EXECUTION.md）

> 配套:`FIX_PLAN.md`(152 个 FX、6 波)、`BEHAVIOR.md`(owner 裁决)、`DEFECTS.md`(175 缺陷)、`AGENTS.md`(硬规矩)。
> 本文件定义**怎么修**:角色分工、并发、单条流水线、多身份评审、自主运行规则。

## 一、角色分工
- **codex(gpt-5.6-sol)= 实现主力**:按我给的收紧规格改代码。**同时只开 1-2 个线程**(高危项 1 个;低危且文件不相交才 2 个)。`--sandbox workspace-write` 直接改工作区。
- **Claude(我)= 总调度 + 评审 + 集成**:
  1. 定根因、写**收紧规格**(精确到文件/函数/边界/验收标准/要跑的守卫);
  2. 派单给 codex、控并发、挑互不相交的文件避免冲突;
  3. codex 交付后**多身份评审**(见三)+ 跑守卫 + 真机/脚本验证;
  4. **合格才合并保留**,不合格打回 codex 重做(带上评审意见);
  5. 维护进度台账,推进下一条。
- 高危(A 档)项:我把规格收得很紧、评审最严;低危(C 档)/机械项:codex 端到端、我快审。

## 二、单条 FX 流水线(7 步)
1. **定位**:我读相关代码+守卫,确认根因与爆炸半径。
2. **写规格**:产出 `audits/fixes/FX-XXX-spec.md`(改哪些文件/怎么改/不许动什么/验收/跑哪些守卫)。
3. **派单**:`codex exec --sandbox workspace-write` 执行规格(1-2 并发,文件不相交)。
4. **多身份评审**:对 `git diff` 用 7 个身份逐一过(见三);写 `audits/fixes/FX-XXX-review.md`。
5. **验证**:跑相关 `test:*-guards` + type-check +(动前端则)双端 build;能真机/写脚本验的就验(如 D-176 那样合成数据验)。
6. **裁决**:全绿+评审通过 → **保留**(标 ✅);否则**打回 codex**(附意见)重做,回到步 3。
7. **记账**:更新 `FIX_PLAN.md` 状态与 `EXECUTION_LOG.md`,取下一条。

## 三、多身份评审清单(每条 diff 都过)
- **🏛 架构师**:分层/依赖方向对?复用既有模块而非重造?与仓库既有模式一致?不新增耦合/绕过封装?
- **🗄 数据师/DBA**:schema/迁移正确且可回滚?事务/锁/幂等?无数据丢失或级联误删?查询无新增 N+1/全表?BigInt/金额精度?
- **🔐 安全审计**:鉴权与归属校验在?输入校验?无秘密泄漏到日志/响应?没引入新的 SSRF/注入/越权/开放重定向?
- **✅ 正确性**:真的修到根因(不是掩盖症状)?边界/空值/并发/回滚路径?无明显回归?
- **💰 业务/产品**:与 `BEHAVIOR.md` 里 owner 的裁决**逐字一致**?金额/比例/时机/文案对?
- **🧪 测试/守卫**:相关 `test:*-guards` 通过?这次修复是否需要**补一条新守卫**把行为钉死(尤其金额/权限/口径)?
- **♻️ 回归**:相邻功能未被破坏?client/server type-check 绿?动前端则双端 build 绿 + 过前端守卫?

> 任一身份亮红 → 不合并,打回。金额/权限/删除类改动,数据师+安全+业务三关必须全绿。

## 四、并发与冲突规避
- 默认 **1 个 codex** 跑高危项;仅当两条 FX 的**改动文件完全不相交**时才并行 2 个。
- 我在派单前列出每条 FX 的"预计改动文件",冲突的排队串行。
- codex 直接改工作区;我用 `git diff`/`git stash` 隔离审查;打回时可 `git checkout -- <file>` 回退该条改动(仅限该条自己产生的、我确认要弃的改动)。

## 五、波次顺序(见 FIX_PLAN.md 明细)
波0 安全阻断 → 波1 资金/计费 → 波2 实例/交付/网络/终端/流量 → 波3 各模块规则+清理 → 波4 大功能(逐个立项)→ 波5 P2/P3 打磨。
波内按"文件聚类 + 横切根因优先"排;横切项(SSRF/apiError/trustProxy/硬删级联)先修,一改收敛多处。

## 六、自主运行规则
- **中途不问 owner**:按已定裁决与本方案自主推进 6 波。
- **过程中新发现的问题**:我自己以多身份"讨论"定性→纳入计划(追加 FX 编号)→按流水线修,不停下来问。
- **仍守硬规矩(AGENTS.md)**:不做大重构;表格布局守卫红线不碰;UI 只动样式/文案;**不 commit / 不 push / 不发版 / 不碰生产——这几件永远等 owner 明确指令**。工作区可改、可攒 diff。
- **阶段汇报**:每完成一个波次(或积累若干条)给 owner 一次进度小结,但不阻塞、不等待答复即继续。

## 七、进度台账

- `EXECUTION_LOG.md`:逐条流水——spec 链接、codex 结果、7 身份评审结论、守卫/验证证据、合并裁决。
